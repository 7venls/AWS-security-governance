# IAM とは？

IAM = AWSにおける“入館証と権限証明書”を管理する仕組み

# IAMが決めている2つのこと

### 認証（Authentication）

・あなたは誰？

・ログインできる？できない？

例：

・正しいIDとパスワードか。

・MFA（2段階認証）を通っているか。

### 認可（Authorization）

・あなたは何をしていい？

・S3見ていい？

・EC2触っていい？

IAMの本質は 認可（権限管理）

# IAMの4要素

### ユーザー（User）

・人

・あなた、開発者、運用担当など

ログインする主体

### グループ（Group）

・役割の箱

・複数ユーザーをまとめる

例：

・developers

・operators

・admins

権限は「人」ではなく「役割」に持たせる

### ポリシー（Policy）

・できること／できないことのルール

・JSONで書かれた「許可リスト」

例：

・S3は読むだけOK

・EC2は触っちゃダメ

権限の正体はすべてポリシー

### ロール（Role）

・AWSサービス用のユーザー

・人は使わない

例：

・EC2がS3にアクセスする

・LambdaがDynamoDBにアクセスする

「AWSサービスがAWSを操作するための身分証」

### 人用とサービス用

【人】

ユーザー

 └ グループ
 
     └ ポリシー

【AWSサービス】

EC2 / Lambda

 └ ロール
 
     └ ポリシー
