# AWS とはなにか？（責任の境界で理解する）

## AWS の役割（AWS責任）

AWS が責任を持つのは、主に以下の責任である。

・データセンターの物理セキュリティ

・ハードウェア（サーバ、ストレージ）

・基盤ネットワーク

・クラウド基盤の可用性

AWS は利用者がどのように設定、運用するかまでは保証しない。

## 利用者の責任（利用者設定領域）

AWS を利用する場合、以下は利用者責任となる。

・IAM（誰が何をできるか）

・ネットワークの公開・非公開設定

・データの暗号化有無

・ログの取得・監視

・セキュリティ設定全般

これらが不十分な場合、

AWSを利用していても情報漏えいや不正アクセスは発生する。

## オンプレミスとの違い

・オンプレミス環境では、  

ITインフラのすべてを自社で管理する必要がある。

・AWSでは物理部分の管理はAWSに委ねられるが、  

責任が軽くなるわけではなく、責任の分担が変わるだけである。
