# リージョン / AZ（可用性と監査視点）

## このテーマを学ぶ意味

AWSは「壊れないサービス」ではなく、障害発生を前提に設計することが重要である。

監査の観点では、単一AZ構成は障害発生時にサービスが停止するリスクとして評価される。  

そのため、リージョンとAZの意味を理解し、可用性を確保する設計ができるか説明できる必要がある。

## リージョンとAZとは？

### リージョン

・AWSの物理的な場所（都市・国単位）

　例：東京リージョン、バージニア北部リージョン
 
【監査的意味】

・災害対策やデータ保管場所の統制

・法規制（海外データ保管など）のチェックポイント

### AZ（アベイラビリティゾーン）

・1リージョン内の独立したデータセンター群

・複数AZに跨ることで障害耐性を確保

【監査的意味】

・単一AZのみは「障害時にサービス停止のリスク大」

・マルチAZ構成があるかがチェックされる。

## 単一AZとマルチAZ（概念図）

単一AZ（危険）
[AZ-A] ── EC2 / RDS
障害発生 → 全部停止

マルチAZ（安全）
[AZ-A] ── EC2 / RDS
[AZ-B] ── EC2 / RDS
障害発生 → 別AZで継続
